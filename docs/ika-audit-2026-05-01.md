## 2. Frontend — TYPE-CHECK PASSES, TWO RUNTIME BUGS

```
pnpm exec tsc --noEmit  →  exit 0
```

But there are two bugs that only surface at runtime.

### Bug A (BLOCKER) — Test oracle 404 on `/attest`

**Where:** `app/lib/ika.ts:120` calls `${IKA_ORACLE_URL}/attest`
with `NEXT_PUBLIC_IKA_ORACLE_URL=http://localhost:3000/api/ika-test-oracle`.
The actual route file is `app/api/ika-test-oracle/route.ts`, which serves
`/api/ika-test-oracle` — not `/api/ika-test-oracle/attest`.

**Result:** clicking **Verify Collateral Lock** in `CollateralOnboarding.tsx`
calls `pollOracleAttestation`, which retries until the 5-minute timeout, then
throws `IKA oracle did not confirm lock within 300s`.

**Why type-check missed it:** path strings aren't typed.

**Why backend tests missed it:** the Mocha suite signs the attestation directly
with Node's `crypto.sign` and never hits the Next.js route.

**Fix:** move the route to its `/attest` subpath so the file/URL convention
matches the real `https://oracle.ika.xyz/v1/attest` structure.
File path: `app/api/ika-test-oracle/attest/route.ts`.

### Bug B (BLOCKER) — React rules-of-hooks violation in `CollateralOnboarding.tsx`

**Where:** `components/borrower/CollateralOnboarding.tsx:184-185`.

```tsx
if (!connected) return <…/>;       // early return
if (collateral) return <…/>;       // early return
if (isLoading) return <…/>;        // early return

const [isCreatingDwallet, setIsCreatingDwallet] = useState(false); // ← hook AFTER returns
const [suiSeedHex, setSuiSeedHex] = useState('');                  // ← hook AFTER returns
```

**Result:** the first time the component renders without collateral and the
wallet is connected, React captures these two extra hook slots. The next render
where `collateral` becomes truthy (after attach) takes a different early
return, the hook count drops, and React throws:

> Rendered fewer hooks than expected. This may be caused by an accidental
> early return statement.

**Why type-check missed it:** rules-of-hooks is an ESLint rule, not a TS error.
A `pnpm lint` would have caught it.

**Fix:** hoist both `useState` calls above all early returns.

## 6. Devnet end-to-end is still owed

The `yarn test:skip-build` run uses **localnet** (`solana-test-validator`),
not devnet. To prove the deployed devnet binary is byte-identical, one of:

- Run the same `tests/ika-collateral.ts` against `--provider.cluster Devnet`
  with funded keypairs, OR
- Walk the UI flow on devnet manually (see test plan section).

Both are "nice to have" — the binary built from the same source as the test
suite is what was deployed, so localnet correctness is strong evidence
devnet is correct. But it is not absolute proof.
